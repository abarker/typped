class wff_language.language_data.LanguageData[source]¶

Bases: object

Class for storing, searching, and retrieving data for the Language class. The language elements which have data stored for them are defined by

langElems = (“type”, “typeAlias”, “quant”, “const”, “var”,
“stdFun”, “prefixFun”, “infixFun”)

The program which uses this class is responsible for making sure that any name argument which is passed in is valid in the namespace for the language element it is being added as (“type”, “variable”, etc.) For example, types and variables may or may not share a namespace or have an intersecting namespace (that is a language-design choice). We do assume that no name can ever name two different things (over all the language elements), except for functions which can have the same names if they have different type signatures (provided the function overloading flag is set). Function name overloading is allowed for all the variants of function names.

Given that the names are always valid in their namespaces we can just add a valid name for any sort of language element (types, variables, etc.) provided that that name has not been added (stored) for any of the other language elements. That is, we can iterate over langElems and check that the name is not defined for any of them. This is described further in the next paragraph.

If two language elements, say types and variables, have disjoint namespaces then a valid type name is assumed never to have been added as a variable name in the first place (and vice versa). So it does not hurt to check if the name has been defined for a variable before adding it as a type (and vice versa). If the namespaces intersect, any names in the disjoint regions still have this same property. For names in the intersection of, say, the type and variable namespaces we do need to check that the name has not been defined either for types or for variables before adding it as either a type or a variable. An iteration over langElems takes care of this.

Precedence values passed in can be strings or ints, but they will be converted to ints before being stored.

In using different block scopes with a dot name convention, such as env1.env2.name, the full dotted names should be added or checked, since env1.name is different from env1.env2.name. This module does not keep track of any block-scoping environments.

Note this is an inefficient prototype implementation. It just stores all the data in dictionaries indexed by names. The function dictionaries have lists of function-signature data as values, since function name overloading is allowed.

The types dictionary is the only one which is slightly more complicated, but it just contains TypeData class instances to hold various type attributes.

The LanguageData class also contains a RegexTrieDict, which it adds all defined types to (and deletes if that is allowed). An application can insert any other string/value pairs that it chooses to into the RegexTrieDict (like command-names, whitespace, parens, etc). This allows the data structure to be used for lexical analysis. It will always know the defined names in the language because the LanguageData class updates it whenever it is updated.

Define a type and an optional list of types it can be converted to. Conversion types can be either types or type aliases. The default for conversionTypes is the empty list.

The type typeName cannot have been previously defined in the type namespace or an exception will be raised. If necessary this can be checked with the isAlreadyDefinedType method before attempting the add, or else the exception can be caught assuming that

raiseExceptionOnLanguageDataError[“AlreadyDefinedError”] = False.

class wff_language.language_data.TypeData[source]¶

Bases: object

An internal record for storing data about types in the types dictionary. Users never see instances of this type, and it may change in different implementations.

Only names (name strings) are actually stored in the various field lists. The various other properties associated with a name can then be looked up by using that name as an index into the dict for the corresponding language element.
