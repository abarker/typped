2.1.4. wff_language.external_program_calls module¶

This module contains code involving function calls to external programs. All the system-specific information is also localized here.

wff_language.external_program_calls.callExternalSubprocess(commandList, stdinFilename=None, stdoutFilename=None, stderrFilename=None, env=None)[source]¶

Run the command and arguments in the commandList. Will search the system PATH for commands to execute, but no shell is started. Redirects any selected outputs to the given filename. Waits for command completion.

wff_language.external_program_calls.cleanupAndExit(exitCode)[source]¶

Exit the program, after cleaning up the temporary directory.

wff_language.external_program_calls.convertWindowsPathToCygwin(path)[source]¶

Convert a Windows path to a Cygwin path. Just handles the basic case.

wff_language.external_program_calls.getCanonicalAbsolueExpandedDirname(path)[source]¶

Get the absolute directory name from a possibly relative path.

wff_language.external_program_calls.getCanonicalAbsoluteExpandedPath(path)[source]¶

Get the canonical form of the absolute path from a possibly relative path (which may have symlinks, etc.)

wff_language.external_program_calls.getDirectoryLocation()[source]¶

Find the location of the directory where the module that runs this function is located. An empty directory_locator.py file is assumed to be in the same directory as the module. Note that there are other ways to do this, but this way seems reasonably robust and portable. (As long as the directory is a package the import will always look at the current directory first.)

wff_language.external_program_calls.getExternalSubprocessOutput(commandList, printOutput=False, indentString='', splitLines=True, ignoreCalledProcessErrors=False, env=None)[source]¶

Run the command and arguments in the commandList. Will search the system PATH. Returns the output as a list of lines. If printOutput is True the output is echoed to stdout, indented (or otherwise prefixed) by indentString. Waits for command completion. Called process errors can be set to be ignored if necessary.

wff_language.external_program_calls.getParentDirectory(path)[source]¶

Like os.path.dirname except it returns the absolute name of the parent of the dirname directory. No symbolic link expansion (os.path.realpath) or user expansion (os.path.expanduser) is done.

wff_language.external_program_calls.getTemporaryDirectory()[source]¶

Create a temporary directory and return the name. The caller is responsible for deleting it (e.g., with shutil.rmtree) after using it.

wff_language.external_program_calls.getTemporaryFilename(extension='', useProgramTempDir=True)[source]¶

Return the string for a temporary file with the given extension or suffix. For a file extension like .pdf the dot should also be in the passed string. Caller is expected to open and close it as necessary and call os.remove on it after finishing with it. (Note the entire programTempDir will be deleted on cleanup.)

wff_language.external_program_calls.globIfWindowsOs(path, exactNumArgs=False)[source]¶

Expands any globbing if systemOs is Windows (DOS doesn’t do it). The argument exactNumFiles can be set to an integer to check for an exact number of matching files. Returns a list.

wff_language.external_program_calls.removeProgramTempDirectory()[source]¶

Remove the global temp directory and all its contents.

wff_language.external_program_calls.runExternalSubprocessInBackground(commandList, env=None)[source]¶

Runs the command and arguments in the list as a background process.

wff_language.external_program_calls.samefile(path1, path2)[source]¶

Test if paths refer to the same file or directory.
