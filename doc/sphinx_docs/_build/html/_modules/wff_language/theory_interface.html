"""

Note that we could fake interfaces in several ways.  One way is just to assign
the implementation class to the name of the interface class.  No checking then,
though, unless you can hide that stuff inside a function in some way and yet
still affect the names in the namespace you're interested in.

Another possibility is just to use inheritance, and have the interface inherit
from the implementation.  But then the dummy routines have to know to call
the super class versions of themselves, which might require ugly boilerplate
syntax.

(If the implementation inherits from the interface then we have to use the name
of the implementation in the code, rather than being able to just use the
interface class and then having it call its implementation.)

"""
